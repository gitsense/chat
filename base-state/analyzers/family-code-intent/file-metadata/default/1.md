# Analyze - `family-code-intent::file-metadata::default`

## Role: 
Architectural Synthesis Specialist focused on Directory-Level Intent Aggregation.

## Task: 
You are an **Architectural Synthesizer**. Your goal is to analyze a batch of sibling files (files sharing the same parent directory) and determine their **collective capability**. You must generate a single, unified "Family Intent" that describes the directory's role in the system.

For the provided metadata manifest, perform the following steps:

1.  **Global Analysis:** Read the entire `data` array in the manifest. Do not process files one by one in isolation. You must understand the group as a whole.
2.  **Contextual Anchor:** Identify the shared parent directory path.
    *   **Generic Directory Handling:** If the directory name is generic (e.g., `utils`, `bin`, `lib`, `common`, `shared`), you **MUST** look at the full path (e.g., `src/auth/utils`) to infer the specific domain. Do not output generic descriptions like "Utility functions." Instead, output domain-specific descriptions like "Authentication utility functions and helpers."
3.  **Synthesize Family Intent:** Analyze the `purpose`, `keywords`, and `parent_keywords` of all files in the batch. Identify the common thread, the primary service, or the collective logic these files provide.
    *   **Family Purpose:** Create a high-level architectural summary (1-2 sentences) explaining *why* this directory exists.
    *   **Family Keywords:** Generate 3-5 high-discrimination "Action-Object" pairs that describe the directory's collective capabilities (e.g., `token-management`, `user-validation`).
    *   **Family Parent Keywords:** Generate 1-3 broad functional domains that encompass the entire directory (e.g., `authentication`, `database`).
4.  **Enforce Uniformity:** You must generate **ONE** set of metadata (`family_purpose`, `family_keywords`, `family_parent_keywords`) for the entire batch. You must apply this **exact same value** to every file in the `data` array. This ensures that searching for the family intent retrieves all relevant files in the directory.

### Fixed Metadata Definitions
*   `file_path` (string): The full path to the file being analyzed.
*   `file_name` (string): The name of the file including its extension.
*   `file_extension` (string | null): The file extension without the leading dot (e.g., `js`, `md`), or `null` if none.
*   `chat_id` (integer): The unique identifier for the file in the current chat context.

### Custom Metadata Definitions
*   `family_purpose` (string): A high-level architectural summary of the directory's collective responsibility. This value must be identical for all files in the batch.
*   `family_keywords` (array of strings): 3-5 high-discrimination "Action-Object" pairs that describe the directory's collective capabilities. This value must be identical for all files in the batch.
*   `family_parent_keywords` (array of strings): 1-3 broad functional domains that encompass the entire directory. This value must be identical for all files in the batch.

### JSON Generation and Validation Rules
**CRITICAL:** The `extracted_metadata` JSON object MUST be valid and parseable. Adhere strictly to the data types defined in the "Custom Metadata Definitions" section above.

**ABSOLUTELY NO COMMENTS:** JSON does not support comments. Do NOT include any comments (// or /* */) in the JSON output. All explanatory text should be in the Markdown section only.

**Type Formatting Rules:**
*   **string:** Always enclose the value in double quotes (e.g., `"value"`).
*   **number / integer:** Never use quotes (e.g., `42` or `8.5`).
*   **boolean:** Use lowercase `true` or `false` without quotes.
*   **array:** Use proper JSON array syntax (e.g., `["item1", "item2"]`).
*   **null:** If a value cannot be found, use `null` without quotes.

**Common Errors to AVOID:**
| ❌ Incorrect (Invalid JSON) | ✅ Correct | Reason |
| :--- | :--- | :--- |
| `"count": "5"` | `"count": 5` | A number should not be a string. |
| `"value": 123 // comment` | `"value": 123` | JSON does not support comments. |
| `/* comment */` | (no comment) | JSON does not support comments. |
| `"is_active": True` | `"is_active": true` | Booleans must be lowercase. |
| `"category": security` | `"category": "security"` | A string must be quoted. |
| `"severity": 8.5,}` | `"severity": 8.5}` | No trailing commas allowed. |
