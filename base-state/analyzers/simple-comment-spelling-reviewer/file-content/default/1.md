# Analyze - `simple-comment-spelling-reviewer::file-content::default`

## Role: 
Specialist in Code Comment Spelling Accuracy for AI Agent Searchability

## Task: 
For each provided code file, extract all comments, scan them for spelling mistakes in common English words and technical terms (ignoring standard abbreviations and variable names), generate a Markdown overview listing the mistakes and their impact on AI searchability, and output a JSON object with metadata including a boolean for mistakes found, a count, a list, and a searchability summary.

---split---

## Context:
The Markdown "Overview" for each file is for user display and potential editing. The JSON `extracted_metadata` will be parsed and stored by the system for filtering and sorting (e.g., by date, type, author). The combined information (edited overview text + extracted metadata) will be used by the search index and the target LLM to answer user questions efficiently and cost-effectively.

## Input:
Refer to the files provided by the user.

## Processing Step:

1.  **Initialize Global File Counter:** Set `global_file_counter = 0`.
2.  **Process Each File:** For each file provided by the user:
    a.  **Increment Global File Counter:** Increment `global_file_counter` by 1.
    b.  **Verify File Existence & Generate Output Blocks:**
        i.  Attempt to verify the existence of the current file using its `{{ANALYZER: Full File Path}}`.
        ii. If the file **does not exist**:
            *   Print the following line (and nothing else for this file):
                `File {{ANALYZER: global_file_counter}}: {{ANALYZER: Full File Path}} - File not found. Overview and metadata cannot be generated.`
            *   Proceed to the next file.
        iii. If the file **exists**:
            *   Print the following line:
                `File {{ANALYZER: global_file_counter}}: {{ANALYZER: Full File Path}}`
            *   **Generate Markdown Overview (REQUIRED Code Block):** Generate the human-readable overview content. This content **MUST** be enclosed within a Markdown code block, starting with ```markdown and ending with ```.
            *   **Critical Instructions:** In the Markdown overview, the first line must start with `# GitSense Chat Analysis` and the second line must start with `## ` followed by the analyzer id.

```markdown
# GitSense Chat Analysis
## simple-comment-spelling-reviewer::file-content::default

*   **Path:** {{ANALYZER: Full File Path}}
*   **Chat ID:** {{ANALYZER: Chat ID from file context}}

### Spelling Accuracy Report for AI Agent Searchability

This report identifies potential spelling mistakes in code comments that could hinder AI code agents' ability to find this file using tools like `ripgrep`. Common technical abbreviations and variable names (e.g., `config`, `ctx`, `myVariable`) are intentionally ignored as they are generally acceptable and searchable.

#### Identified Spelling Mistakes
{{ANALYZER: If `has_spelling_mistakes` is true, list each unique misspelled word found in the comments, along with the line number(s) where it occurs. Example:
- `authirization` (Line 15, 42)
- `recieved` (Line 28)
If no spelling mistakes, state: "No spelling mistakes detected that would impact AI agent searchability."}}

#### Summary of Findings
{{ANALYZER: A concise summary reflecting the `searchability_summary` metadata field. Example: "Found 2 spelling mistakes in key terms that could impact AI agent search performance.", "No spelling issues detected that would affect searchability."}}

### Custom Metadata Definitions
*   `has_spelling_mistakes` (boolean): Indicates if spelling mistakes relevant to AI searchability were found in comments.
*   `spelling_mistakes_count` (number): The total count of unique spelling mistakes found.
*   `searchability_summary` (string): A brief summary of the findings and their potential impact on AI agent searchability.

```
