# Code Smarter 101

**Coding not required.**

In this 101 guide, we will show you how to turn your repository into shareable, queryable intelligence. You will learn how to define patterns, run analysis at scale, and package the results so your agents can find answers that standard search cannot.

### Step 0: Import a Repository

Before we can create intelligence, we need to import the code we want to make smarter. To understand how, let's practice with `ripgrep`.

```bash
# Clone the repository
git clone https://github.com/BurntSushi/ripgrep.git
cd ripgrep

# Import it into GitSense Chat
gsc app import git --owner BurntSushi --repo ripgrep --branch master
```

Once imported, the repository will appear in the Repositories Tree on the right pane. The files can now be selected for chat or analysis.

## Try It Now

You now know how to import files. Later, we will show you how to process them into intelligence. But for now, let's see what that intelligence looks like in action.

The repository we'll play with, `smart-ripgrep`, comes with a pre-built Implicit TODOs manifest.

```bash
# Step 1: Clone the repository
git clone https://github.com/gitsense/smart-ripgrep
cd smart-ripgrep

# Step 2: Create a queryable database with the shipped manifest
gsc manifest import implicit-todos

# Step 3: Confirm the 'implicit-todos' brain has been created
gsc brains
```

Now start your coding agent (Claude Code, Cursor, Aider, etc.) and tell it to execute:

```
gsc experts init
```

This command creates a context file that teaches your agent how to query this repository's intelligence.

Now ask your agent:

> "Can you tell me what files currently have implicit TODOs?"

Your agent is answering from analysis that already exists in the repository, not by re-reading files and trying to infer the answer on the fly. The analysis found comments where a developer described a possible problem without tagging it `TODO`. Standard `grep TODO` misses those comments because the keyword is not there.

This is the key difference: GitSense intelligence is already there. Your agent queries it instead of rediscovering it.

## Closing Thought

Coding smarter does not mean typing faster. It means spending less time rediscovering context that already exists somewhere in the codebase or team.

Start with one question your team keeps answering by hand. Build an Analyzer for it, run it across a repository, and let your agent query the result. Package that knowledge, ship it with the code, and your repository becomes a self-serve intelligence layer that travels with your data.
