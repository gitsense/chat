# New Analyzer Requirements

I can help you build and create a new analyzer configuration. To get started, please tell me about the analyzer you want to create.

### Guiding Questions:

1.  **What is the primary purpose of this analyzer?** (e.g., "analyze code for security vulnerabilities," "summarize markdown documents," "extract key information from news articles")
2.  **What specific information should it look for and extract?** (This will help define the task and any custom metadata.)
3.  **How should it present the findings in the human-readable Markdown overview?**
4.  **Which of the extracted metadata fields would you like to see displayed in a progress table?**
5.  **Will your analyzer require any optional reference files?** (e.g., a `README.md`, a style guide, or a configuration file).
    *   **If yes:**
        *   **A. How should the analyzer identify this file?** (By a specific path like `docs/style-guide.md`, or by a pattern like `README.md` or `*.config.json`?)
        *   **B. What is the specific purpose of the file?** (e.g., "Use the content to define the project's context," or "Use the rules to validate code comments.")
        *   **C. What should the analyzer do if a required reference file is missing?** (Stop and report an error, or continue with a warning/default analysis?)

---

### Key Requirements for Your Analyzer

Please keep these rules in mind as we build your analyzer. I will handle the implementation details.

*   **Mandatory Code Blocks:** The analyzer you design will be required to output its findings in two separate code blocks: one for the Markdown overview and one for the JSON metadata.
*   **Analyzer ID Format:** The unique name for your analyzer must only contain letters, numbers, dashes, or underscores (e.g., `my-custom-analyzer`).
*   **Handling Missing Data:** If any piece of metadata cannot be found for a specific file, the analyzer will include that field in the JSON output with a value of `null`. It will not omit fields.
*   **Error Handling:** If a file is corrupted or cannot be read, the analyzer will output a specific error message for that file and continue to the next one.
*   **Reference Files:** If your analyzer requires reference files (like a style guide or README), you will need to provide them in a separate context message starting with `## REFERENCE FILE CONTENT` when you run the analyzer. The analyzer will use these files for context but will not analyze them as part of the main task.
